Introduction

About this Guide

This guide gives a very brief overview on the things that one needs to know to write a mechanism for the SASLv2 API (and thus Cyrus SASLv2). Note that this page is a brief overview only and that the authoritative docuementation are the header files included in the SASL distribution. If you have any questions, please feel free to contact the Cyrus development team at cyrus-bugs@andrew.cmu.edu or the cyrus-sasl mailing list at cyrus-sasl@andrew.cmu.edu .

Please note that this guide is only intended for developers looking to write mechanisms for the SASLv2 API, and that application programmers should be reading this document instead.

What is SASL?

A description of SASL is covered in detail in the programmer's guide, which mechanism developers should probabally read first anyway to become familliar with development using the SASL library.
